ALL CRITERIA USED FOR FAKE BANK ALERT DETECTION (TEXT & IMAGE OCR)

The backend system mathematically analyzes text and images using the following strict criteria:

1. MISSING AVAILABLE BALANCE (+3 Penalty)
Real Nigerian bank alerts always show the new available balance. If the alert completely omits the "Available Balance", it is heavily penalized.

2. SUSPICIOUS MASKED ACCOUNT NUMBER (+2 Penalty)
Official masked account numbers always display the exact last 4 digits (e.g., ****1234). If the account shows 3 or fewer digits at the end (e.g., ****338), the system flags it as dynamically generated by a fake app.

3. BAD COMMA & DECIMAL MATH FORMATTING (+4 Penalty)
Bank software enforces strict grammatical syntax for money. If an amount over 1,000 NGN lacks correct comma groupings or exact 2-decimal trailing zeroes (e.g., "50000.0" instead of "50,000.00"), it is flagged as human-typed fake math.

4. LAZY / SHORT TRANSACTION REFERENCE (+3 Penalty)
Real bank receipts carry massive 15 to 30 character Transaction IDs at the bottom. If the reference string is entirely missing or under 10 characters (e.g., "Ref: 345"), it gets immediately penalized.

5. MISSING OR INVALID AMOUNT (+3 Penalty)
If the algorithm cannot parse any monetary value out of the text/image at all, it assumes the image structure is completely broken and flags it.

6. UNRECOGNIZED BANK NAME (+1 Penalty)
If the name of the bank found in the alert does not match an internal whitelist of 30+ officially recognized Nigerian financial institutions, it receives a minor penalty.

7. SCAM & PHISHING PHRASES (+4 Penalty)
Fake alerts uniquely use amateur phrases like "you have been credited". The system scans for a hardcoded list of confirmed scam phrases and adds a huge penalty for them.

8. URGENT / DEMANDING TONE (+5 Penalty)
Banks do not have emotions. Scammers use words like "urgent", "click here", or "call now" to force victims into action. If an aggressive tone is sensed, a massive penalty drops.

9. SPELLING MISTAKES (+2 Penalty)
If banking-standard vocabulary is badly misspelled (e.g., "alret" instead of "alert"), the system flags it.

10. OPENAI (CHATGPT) OVERRIDE MODULE
In addition to the mathematical Regex rules above, the system passes the text directly to ChatGPT/OpenAI models. If the highly capable AI analyzes the syntax and believes the entire context of the message implies a genuine, legitimate human text that accidentally failed a math rule, it violently overrides all the penalties and resets the alert's status back to 'Real'.
